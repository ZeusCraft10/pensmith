## Critical Anti-Patterns (do NOT repeat these)

- **Lock inside `.paper/`** — OneDrive will eat it. Lock MUST live in `LOCAL_DATA_DIR` (D-08).
- **load OUTSIDE the lock for a mutator** — two concurrent calls would each see the same pre-write value. The W11 10-disjoint-id test would have caught this; W12 needs an analogous regression gate.
- **Skip `LibrarySchema.parse` / `CheckpointSchema.parse` before atomicWriteFile** — defense-in-depth is non-negotiable. Three save-side guards + one load-side guard.
- **Computing your own sha256 of the lock-key** — W3 does it internally. Pass the absolute file path directly.
- **Smuggling later-phase work into Phase 1** — no checkpoint replay logic, no section state machine integration, no /pensmith CLI. Foundation slice ONLY.
