| section sources/draft input → transport | write's drafter input crosses to the LLM; assertDrafterInput validates the shape first (WRTE-04) |
| missing-key state → CLI exit | a key-less run must surface as a non-zero exit, never a silent ok:true placeholder |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-11-05 | Information Disclosure | API-key value leaking via verb-level logging | mitigate | getProviderApiKey is used only as a presence probe + by complete() internally; no verb binds the key to a logged variable; complete() owns the no-leak header path (T-01-07). |
| T-11-06 | Tampering | Raw (unredacted) PII reaching the transport from intake | mitigate | intake passes egressSeed (post-redaction when piiRedact on), never rawAnswers; a call-site comment + grep acceptance criterion enforce it (Pitfall 3). |
| T-11-07 | Tampering | Unvalidated drafter input reaching the LLM (bypassing WRTE-04) | mitigate | assertDrafterInput runs BEFORE complete() in writeOneSection; grep acceptance criterion asserts the ordering (invariant 3). |
| T-11-08 | Repudiation | Silent ok:true on missing key (GEN-06 regression) | mitigate | Every no-key path returns ok:false + process.exitCode=1; grep asserts no `ok:true` on the missing-key branch (Pitfall 4). |
