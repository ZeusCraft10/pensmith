<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| citekey DOI → outbound HEAD | DOI value drives an outbound HTTPS request (SSRF surface) |
| directory basename → parseSectionDirName | Filesystem-derived name parsed into path components |
| report input object → YAML frontmatter | Compile pipeline data serialized to disk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | SSRF / Spoofing | freshness DOI HEAD | mitigate | Validate DOI format via doi.ts before request; HEAD targets `doi.org` only, never an arbitrary URL; 10s timeout + 1 retry cap |
| T-04-06 | Tampering | parseSectionDirName basename | mitigate | Reject `..`, absolute paths, null bytes; slug validated via validateSlug (V12 ASVS) |
| T-04-07 | Information disclosure | freshness probe value leak | accept | Probe sends no request body; only emits status codes to VERIFICATION.md — no secret surface |
| T-04-08 | Tampering | COMPILE-REPORT forward-incompat | mitigate | `schema_version: z.literal(1)` + strict zod (no passthrough) refuses forward-incompat keys (ARCH-07) |
| T-04-09 | Tampering | freshness escalated to a block | mitigate | Pass 1 blocking verdict path untouched; freshness is WARN-only by construction (D-10 / PRD §14) |
| T-04-SC | Tampering | cassettes / npm installs | mitigate | No new deps; cassettes git-committed, header-scrubbed, size-capped; CI `npm run check` runs them |
</threat_model>
