<objective>
Build the compile pipeline — the phase's keystone. `runCompile` acquires a whole-pipeline lock on `.paper/.compile.lock`, then: (1) loads sections in OUTLINE order, checks each section's `verified_against_draft_hash` against `computeDraftHash(currentDraftBytes, sources)`, auto-re-verifies stale sections with Pass 1+3 (and REFUSES on any FABRICATED/MIS-CITED/NOT_FOUND), (2) concatenates draft texts in outline order, (3) runs N-1 per-boundary smoothing calls with citation-token placeholder substitution + post-call token-set equality (reject→raw-concat fallback on drift), runs the deterministic consistency scan, regenerates `.paper/CITATIONS.bib`, and (4) emits `.paper/DRAFT.md` + `.paper/COMPILE-REPORT.md` (schema v1) — all writes atomic. Section files are READ-ONLY throughout. Finally, extend the tier-contract test with compile/revise/write-wave parity cases.

Purpose: COMP-01 (refuse on bad citation), COMP-02 (outline-order concat), COMP-03 (N-1 smoothing + token protection), COMP-04 (atomic writes), COMP-05 (bib regen), COMP-06 (COMPILE-REPORT schema v1), COMP-07 (consistency warnings non-blocking), ARCH-20 (read-only on section files). Depends on Plan 01 (parseOutline, citation-token) + Plan 02 (renderCompileReport, freshness-aware Pass 1). Wave 3.
Output: `bin/lib/compile.ts`, `draft-hash.ts`, `consistency-scan.ts`, `bin/cli/compile.ts`, `workflows/compile.md`, `templates/prompts/smoother.md`, prompt-loader + repo-files patches, 8 test files, tier-contract extension, 3 cassettes.
