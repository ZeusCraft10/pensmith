<objective>
Build the compile pipeline — the phase's keystone. `runCompile` acquires a whole-pipeline lock on `.paper/.compile.lock`, then: (1) loads sections in OUTLINE order, checks each section's `verified_against_draft_hash` against `computeDraftHash(currentDraftBytes, sources)`, auto-re-verifies stale sections with Pass 1+3 (and REFUSES on any FABRICATED/MIS-CITED/NOT_FOUND), (2) concatenates draft texts in outline order, (3) runs N-1 per-boundary smoothing calls with citation-token placeholder substitution + post-call token-set equality (reject→raw-concat fallback on drift), runs the deterministic cross-section claim-consistency scan (flags only — COMP-04) and the citation-density computation vs. the discipline preset target (warn-only — COMP-05), regenerates `.paper/CITATIONS.bib` (D-19 bibtex chokepoint), and (4) emits `.paper/DRAFT.md` + `.paper/COMPILE-REPORT.md` (schema v1, D-14 body sections) — all writes atomic via the D-07 sole-writer chokepoint. Section files are READ-ONLY throughout. Finally, extend the tier-contract test with compile/revise/write-wave parity cases.

It is intentionally NOT split: the keystone compile pipeline must land as one coherent unit, and the Task 1a/1b safety-first split already guarantees the COMP-01 refuse-gate test lands on disk first.
