<objective>
Wire the Plan 01 wave scheduler into the `/pensmith write` flow. When invoked without a section number, `write` loads the outline, builds the wave graph, and drains waves one at a time — running each wave's sections in bounded parallel (Tier 1, `--max-parallel 5`) or serially (Tier 2, forced `--max-parallel 1` with a WARN) — by calling the EXISTING per-section writer for each node. The single-section `pensmith write <n>` path is unchanged. Within-wave failures don't cancel siblings; transitive-dep failures block only the dependent subtree (D-03).

Purpose: ARCH-19 (apply the bounded-parallel primitive to multi-section writing), ARCH-20 (orchestrator persists no wave state). Depends on Plan 01 (scheduler + outline-parse). Runs in Wave 2 alongside Plan 04.
Output: `bin/lib/write-orchestrator.ts`, patched `bin/cli/write.ts`, patched `workflows/write.md`, 2 new test files + a tier-contract registry entry for wave-mode write (CONTRIBUTING.md D-24 — the workflow body's tier-contract obligation is satisfied in THIS plan, where workflows/write.md is changed).
</objective>
