**Atomic write — D-07 LOCKED chokepoint (REVIEWS CONVERGENCE — Codex HIGH "atomic-write fsync claim mismatch"):**

`bin/lib/atomic-write.ts` (locked in Plan 03) is THE ONLY module that performs durable atomic writes in pensmith. It internally does: `writeFile(`${path}.tmp`, content) → open(`${path}.tmp`) → fsync(fd) → close(fd) → rename(`${path}.tmp`, path) → fsync(dir)`. The dir-fsync is required on POSIX for the rename to be durable across power loss.

handoff.ts MUST delegate to this — it does NOT reimplement the pattern:

```typescript
import { atomicWrite } from './atomic-write.js';
// ...
await atomicWrite(HANDOFF_PATH, content);
```

ESLint rule from Plan 03 (no-restricted-imports / chokepoint enforcement) blocks any other module from importing `node:fs/promises` `writeFile + rename` pair — atomic-write.ts is the single source of fsync truth.

**YAML pitfall (RESEARCH #7) — REVIEWS CONVERGENCE (Gemini/OpenCode MEDIUM "frontmatter helper key-deletion preservation"):**

The naïve pattern `JSON.parse(JSON.stringify(yamlObject))` loses comments + key order. But ALSO: `doc.toJSON() → mutate JSON projection → for (const [k,v] of Object.entries(mutated)) doc.set(k,v)` is ALSO LOSSY for key deletion (the deleted key is still present in `doc` after re-setting from the mutated projection — `Object.entries` only iterates kept keys, so deleted keys are never `doc.delete`-d).

Correct pattern: mutate the LIVE `Document` directly via `doc.set(k, v)` and `doc.delete(k)`. The mutator receives a Proxy wrapping the Document so the consumer-facing API is still `(fm) => { fm.foo = 1; delete fm.bar; }`, but every Proxy operation routes through `doc.set` / `doc.delete` against the live Document — so comments + key-order + deletion all survive.

       ```typescript
       import { parseDocument, type Document } from 'yaml';

       const FRONTMATTER_RE = /^---\r?\n([\s\S]*?)\r?\n---\r?\n?([\s\S]*)$/;

       export function parseFrontmatter(text: string): { frontmatter: Record<string, unknown>; body: string } {
         const match = FRONTMATTER_RE.exec(text);
         if (!match) return { frontmatter: {}, body: text };
         const doc = parseDocument(match[1]);
         return { frontmatter: doc.toJSON() ?? {}, body: match[2] };
       }

       export function serializeFrontmatter(frontmatter: Record<string, unknown>): string {
         const doc = parseDocument('');
         for (const [k, v] of Object.entries(frontmatter)) {
           doc.set(k, v);
         }
         return `---\n${doc.toString()}---\n`;
       }

       /**
        * Round-trip safe AND deletion-safe: preserves comments + key order, AND
        * supports key deletion via `delete fm.key`. The mutator receives a Proxy
        * whose every `set`/`deleteProperty` call routes through the LIVE yaml
        * Document API (doc.set / doc.delete) — NOT through a JSON projection.
        *
        * Why a Proxy (REVIEWS CONVERGENCE — Gemini/OpenCode MEDIUM
        * "frontmatter helper key-deletion preservation"):
        * The naïve pattern `for (const [k,v] of Object.entries(fm)) doc.set(k,v)`
        * after a JSON-projection mutate is broken: deleted keys never appear in
        * Object.entries(fm), so doc.delete(k) is never called and the key
        * survives in the serialized output. The Proxy routes deleteProperty
        * directly through to doc.delete(k) on the live Document.
        */
       export function updateFrontmatter(
         text: string,
         mutator: (fm: Record<string, unknown>) => void,
       ): string {
         const match = FRONTMATTER_RE.exec(text);
         const doc: Document = match ? parseDocument(match[1]) : parseDocument('');
         const body = match ? match[2] : text;

         const proxy = new Proxy({} as Record<string, unknown>, {
           get(_t, prop): unknown {
             if (typeof prop !== 'string') return undefined;
             return doc.get(prop);
           },
           set(_t, prop, value): boolean {
             if (typeof prop !== 'string') return false;
             doc.set(prop, value);
             return true;
           },
           deleteProperty(_t, prop): boolean {
             if (typeof prop !== 'string') return false;
             doc.delete(prop);  // live-Document deletion — comments adjacent to surviving keys survive
             return true;
           },
           has(_t, prop): boolean {
             if (typeof prop !== 'string') return false;
             return doc.has(prop);
           },
           ownKeys(): string[] {
             const json = doc.toJSON() ?? {};
             return Object.keys(json);
           },
           getOwnPropertyDescriptor(_t, prop): PropertyDescriptor | undefined {
             if (typeof prop !== 'string') return undefined;
             if (!doc.has(prop)) return undefined;
             return { enumerable: true, configurable: true, value: doc.get(prop) };
           },
         });

         mutator(proxy);

         return `---\n${doc.toString()}---\n${body}`;
       }
       ```
