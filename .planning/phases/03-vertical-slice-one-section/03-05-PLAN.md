    - Role: Outline architect
    - Inputs: {{topic}}, {{length}}, {{candidateSources}} (array of SourceCandidate)
    - Task: Propose 3-7 sections, each with: number, slug (kebab-case), title, depends_on (array of slugs), estimated_word_count
    - Hard constraint: depends_on must reference sections by slug (D-03)
    - Hard constraint: NO self-reference in depends_on (D-04)
    - Output: YAML matching PlanFrontmatter shape (top-level: list of sections)

    **plan-section.md** (PLAN-01):
    - Role: Section planner
    - Inputs: {{section}} (number, slug, title, depends_on), {{candidateSources}}
    - Task: Assign 3-15 SourceCandidate citekeys to assigned_sources, write a 100-200 word section brief
    - Hard constraint: assigned_sources MUST be citekeys that exist in candidateSources (no inventions)
    - Output: PLAN.md frontmatter YAML + a "## Brief" Markdown section

    **write-section.md** (WRTE-01/03, D-21, D-13 — calibrated):
    - Role: Section drafter
    - Inputs: {{section}}, {{brief}}, {{assignedSources}} (array of SourceCandidate, restricted to this section's mapped sources only — PRD §7.6)
    - Task: Write the section in Markdown using Pandoc [@citekey] tokens for citations
    - Hard constraint: NEVER invent a DOI, author name, year, or journal. Every citation MUST be `[@<citekey>]` where citekey is in assignedSources.
    - Hard constraint: NEVER use `[1]` or `(Author, 2024)` style — Pandoc tokens only (D-21)
    - Hard constraint: If a claim cannot be supported by assignedSources, write it WITHOUT a citation rather than fabricating one (the orphan-claim verifier will flag it later — D-13)
    - Output: Markdown body only (no frontmatter; the workflow wraps it)

    **verify-pass1.md** (VRFY-01):
    - Role: Citation integrity narrator (the DETERMINISTIC fuzzy gate did the work; you describe the failures in plain English for the VERIFICATION.md report)
    - Inputs: {{pass1Results}} — array of { citekey, doi, claimedTitle, claimedAuthor, foundTitle, foundAuthor, titleJW, authorJW, verdict }
    - Task: For each MIS-CITED or FABRICATED entry, write 1-2 sentences explaining what went wrong (low title match? DOI registrar missing? author transposition?)
    - Hard constraint: NEVER override the deterministic verdict; only narrate it (D-13)
    - Output: Markdown subsection "## Pass 1 — Citation Integrity" with a verdict table + per-failure narrative

    **verify-pass3.md** (VRFY-04/05):
    - Role: Quote integrity narrator (same — deterministic Levenshtein gate did the work)
    - Inputs: {{pass3Results}} — array of { quote, doi, levRatio, verdict }
    - Task: For each NOT_FOUND entry, write 1-2 sentences (suspected distortion? quote too generic?)
    - Hard constraint: NEVER override (D-13)
    - Output: Markdown subsection "## Pass 3 — Quote Integrity"

    **draft-summary.md**:
    - Role: Section draft summarizer (for HANDOFF.json resumePrompt)
    - Inputs: {{draftMarkdown}}, {{sectionTitle}}
    - Task: Write a ≤300 character resumePrompt summarizing what was written + what's next
    - Hard constraint: ≤300 characters — measured by Buffer.byteLength('utf8'). The HANDOFF schema caps the field at 1000 chars but pointers-not-content (D-18) wants this much tighter.
    - Output: Plain text, no Markdown formatting

    For each prompt, prefer concrete examples over abstract instructions. Each prompt should be 30-80 lines.
