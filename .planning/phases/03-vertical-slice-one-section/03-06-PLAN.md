    ## Body

       - Build pass1Results array.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Workflow body → bin/cli/<verb>.ts | Workflow is the prompt; the verb is the executor. The body must not bypass the verb's input validation (e.g. assertDrafterInput). |
| Workflow → other sections | Each per-section verb (plan/write/verify) is invariant-bound to only touch its target sectionDir. |
| LLM narrator → deterministic verdict | verify-pass1/pass3 prompts MUST NOT alter the verdict; the prompt only narrates. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-08 | Tampering | Workflow verb mutates wrong section | mitigate | TEST-09 mtime invariant (section-isolation.test.ts) enforces in CI; per-verb sectionDir(n,slug) is the only filesystem write target. |
| T-3-10 | Tampering | Drafter input contract bypass (workflow passes extra fields, leaking info) | mitigate | write.md calls assertDrafterInput; drafter-input.ts (Plan 07) throws on superset. |
| T-3-VERDICT-01 | Repudiation | LLM narrator overrides deterministic verdict | mitigate | verify-pass1.md and verify-pass3.md prompt include "NEVER override the deterministic verdict" hard constraint (D-13, Plan 05). Verify verb computes verdict BEFORE prompt invocation; prompt only formats narrative. |
| T-3-APPROVAL-01 | Repudiation | Outline written without user confirmation | mitigate | outline.md APPROVAL GATE pauses unless --yolo; PRD non-negotiable. |
| T-3-PII-01 | Information Disclosure | PII from intake answers persisted unredacted | mitigate | intake.md body invokes PII redactor before persisting INTAKE.md; tests/pii.test.ts contract. |

</threat_model>
