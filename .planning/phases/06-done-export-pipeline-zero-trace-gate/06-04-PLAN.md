<objective>
Implement `bin/lib/exporter.ts` — the export module that satisfies the LOAD-BEARING zero-trace contract. `zeroTracePatch()` is the mandatory last step of every .docx export: it loads the ZIP in-process with JSZip, blanks every identifying field in docProps/core.xml + docProps/app.xml, epochs the timestamps, and writes back atomically — guaranteeing zero 'pensmith' trace and zero non-empty metadata field. `exportDraft()` orchestrates Pandoc shellout for .docx/.pdf/.tex/.md, falls back to markdown-only with a banner when Pandoc/PDF-engine is absent (never throwing), and copies .paper/CITATIONS.bib alongside the output (DONE-08).

Purpose: DONE-07 + TEST-10 are THE gating non-negotiable (CLAUDE.md: "No exported-document trace ... THE gating test"). Pandoc alone is insufficient — even `--metadata author=""` leaves the empty-but-present field, and a source YAML header can inject a real name; the in-process ZIP patch is the only deterministic guarantee. Because Pandoc + the PDF engine are ABSENT on the build machine, the zero-trace test (06-01) runs zeroTracePatch against a committed fixture .docx — this plan's zeroTracePatch MUST make that offline test GREEN. exportDraft must recoup the isPandocPresent() check at call time (Pitfall 1) so a missing Pandoc degrades to md-only rather than crashing.
Output: One production module turning RED tests/zero-trace-export.test.ts AND tests/exporter.test.ts from Wave 0 GREEN (the offline, Pandoc-free paths).
</objective>
