Close HARD-01: canonicalize lock keys inside stubFor so two callers targeting the same underlying file via different path conventions always share one lock. This fixes the silent re-enablement of the BLOCKER-01/02 clobber races — the macOS /var→/private/var hazard surfacing as a real production bug.

The fix is single-site canonicalization in stubFor plus removing the now-harmful 'compile:' prefix.
Output: lock.ts stubFor canonicalizes before hashing; compile.ts drops its prefix.
</objective>
