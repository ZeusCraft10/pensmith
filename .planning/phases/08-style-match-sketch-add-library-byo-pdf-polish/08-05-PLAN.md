<objective>
Wire style-match end-to-end (STYL-03 + STYL-04), register papers in the global library at intake with a class (LIB-04). This plan delivers BOTH ends of the style-match loop: (1) the PRODUCER — an opt-in `intake --style-samples` path that BUILDS `.paper/STYLE.json` from the user's writing samples and SURFACES cross-paper reuse unconditionally; and (2) the CONSUMER — the drafter integration that adds an optional styleProfilePath to the strict DrafterInputSchema and resolves voiceHint by a strict priority where a section's PLAN.md voice_hint always overrides the style-match render. Intake global-library registration is a non-fatal side effect that records the paper's id/name/folderPath/class and SEEDS status:'intake'; the live lifecycle status is DERIVED by `list` (08-01) at display time, so intake does NOT chase status across later verbs. The README section uses honest framing only.

Purpose: Close the style-match loop FULLY — opt-in profile build (producer) → STYLE.json on disk → drafter consumes it (consumer) — plus the library lifecycle entry point (intake → global registry) and the sensitive-feature transparency disclosure. Covers SC-2 (STYLE.json is produced by a real verb path AND consumed by the drafter; cross-paper reuse surfaced; README disclosure) and SC-1 (class assignment + folderPath at intake, which is what makes the DERIVE-AT-DISPLAY status work).
Output: opt-in style-match producer in intake, additive schema field, voiceHint resolution in write.ts, intake registration, README section + content guard.
