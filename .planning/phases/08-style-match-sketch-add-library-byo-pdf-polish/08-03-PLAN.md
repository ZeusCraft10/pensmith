<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PDF bytes → python3 subprocess | untrusted PDF content reaches an external interpreter via a tmpfile |
| pdf-parse version → import | a lockfile refresh could drift the exact pin and break the sub-path import |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-03-01 | Tampering | pymupdf shellout command injection | mitigate | execFile arg array (NOT a shell string); tmpfile path is internally generated and JSON.stringify-embedded — no user-controlled shell metacharacters reach the interpreter |
| T-08-03-02 | Denial of Service | runaway / hung python3 | mitigate | execFile timeout (15s) + maxBuffer 10MB; any timeout/overflow → caught → null |
| T-08-03-03 | Denial of Service | subprocess spawn per PDF | mitigate | Fallback runs ONLY in the image-only near-empty branch, never on healthy PDFs (Pitfall: per-PDF spawn) |
| T-08-03-04 | Tampering | pdf-parse version drift | mitigate | Exact pin 1.1.1 in package.json + repo-files assertion that require('pdf-parse/package.json').version==='1.1.1' (Pitfall 4) |
| T-08-03-05 | Denial of Service | absent pymupdf crashing ingestion | mitigate | pymupdfShellout returns null on ANY failure (never throws); extractPdfText degrades to near-empty pdf-parse text + WARN (Pitfall 5) |
| T-08-03-06 | Information Disclosure | tmpfile leak of PDF bytes | mitigate | finally unlink (errors ignored); tmpfile in os.tmpdir() with a random suffix |
| T-08-03-SC | Tampering | npm/pip/cargo installs | accept | No npm installs; pymupdf is a Python shellout only (08-RESEARCH audit — never imported as a package) |
</threat_model>
