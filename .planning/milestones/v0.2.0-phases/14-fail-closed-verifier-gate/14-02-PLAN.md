## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| section dir (filesystem) → compile refuse-gate | an attacker/mistake deletes or truncates a VERIFICATION.md to bypass the gate |
| VERIFICATION.md text → parseVerdictRows | a tampered file may craft verdict-like rows to confuse the parser |
| verify.ts writer ↔ compile.ts parser | format drift between two copies silently nulls the blocking set |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Tampering | compile GATE-01 (absent/empty VERIFICATION.md) | mitigate | hasStatus = /^Status:\s*\S/m guard refuses absent/empty/no-Status before the verdict parse; continue skips the section; structurally unbypassable (collect-all-then-refuse) |
| T-14-02 | Tampering | verdict writer↔parser drift (GATE-02) | mitigate | one shared verdict-rows.ts; round-trip + mutation test (Plan 01) breaks on any future format change instead of silently returning [] |
| T-14-02-INJ | Tampering | crafted verdict row in a tampered VERIFICATION.md (V5 input validation) | mitigate | parseVerdictRows regex is anchored '^\s*-\s*', citekey grammar locked [a-z][a-z0-9_-]*, verdict must be in BLOCKING_VERDICTS; freshness table rows excluded by the list anchor (Pitfall 2) |
| T-14-01-UNVERIF | Repudiation | over-refusing legitimate 'Status: unverifiable' sections | accept | Pitfall 3 — hasStatus matches 'Status: unverifiable'; those sections proceed with zero verdict rows (existing, in-scope-excluded behavior); regression-tested |
