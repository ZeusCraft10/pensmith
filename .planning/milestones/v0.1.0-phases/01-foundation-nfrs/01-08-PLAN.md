  <action>
    Implement bin/lib/pii.ts per D-49. Required exports: `redactPii`, `redactKeys`, `classifyPii`, plus types `PiiKind` and `PiiMatch`.

    **Regex specifications (per D-49, hand-rolled — do NOT add a regex library):**

    1. **EMAIL** — `/[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}/g`. Standard local-part + dotted domain; intentionally not RFC-strict (D-49: "good enough for redaction, false positives acceptable, false negatives are the failure mode").
    2. **PHONE_US (kind: PHONE)** — `/(?:\+?1[-.\s]?)?\(?\d{3}\)?[-.\s]?\d{3}[-.\s]?\d{4}/g`. Optional country code, optional parens around area code, optional separators (dash/dot/space).
    3. **SSN** — `/\b\d{3}-\d{2}-\d{4}\b/g`. Word-boundary anchored; no spaceless form (per D-49: spaceless 9-digit strings have too high false-positive rate; only the dashed canonical form).
    4. **NAME** — `/\b[A-Z][a-z]{1,20}(?:[ -][A-Z][a-z]{1,20}){1,2}\b/g`. Two-or-three capitalized tokens (handles middle name and hyphenated surnames). Length cap 21 chars per token to avoid pathological backtracking.
    5. **DATE** — union of three sub-patterns:
       - ISO: `/\b\d{4}-\d{2}-\d{2}\b/g`
       - US: `/\b(?:0?[1-9]|1[0-2])\/(?:0?[1-9]|[12]\d|3[01])\/(?:19|20)\d{2}\b/g`
       - EU: `/\b(?:0?[1-9]|[12]\d|3[01])\.(?:0?[1-9]|1[0-2])\.(?:19|20)\d{2}\b/g`

    **classifyPii implementation:**
    - Run each regex with `String.prototype.matchAll`.
    - Collect `{kind, span:[m.index, m.index+m[0].length], raw: m[0]}`.
    - Sort by `span[0]` ascending.
    - Resolve overlaps: when two matches overlap, keep the one with the longer raw; on tie keep the earlier-starting one. Never produce overlapping spans in output.

    **redactPii implementation:**
    - Compute spans via classifyPii.
    - Walk spans in REVERSE order (right-to-left) and splice the original string: `out = out.slice(0, span[0]) + tag + out.slice(span[1])` where tag is `[REDACTED:${kind}]`.
    - Walking right-to-left avoids span-recomputation as offsets shift.

    **redactKeys implementation:**

    ```typescript
    const SENSITIVE = new Set([
      'authorization','x-api-key','api_key','apikey','token','access_token',
      'refresh_token','secret','client_secret','cookie','set-cookie',
      'password','passwd','ssn','ssn_last4',
    ]);

    function isPlainObject(v: unknown): v is Record<string, unknown> {
      if (v === null || typeof v !== 'object') return false;
      const proto = Object.getPrototypeOf(v);
      return proto === Object.prototype || proto === null;
    }

    function clone(v: unknown): unknown {
      if (Array.isArray(v)) return v.map(clone);
      if (isPlainObject(v)) {
        const out: Record<string, unknown> = Object.create(null);
        for (const k of Object.keys(v)) out[k] = clone((v as any)[k]);
        return out;
      }
      return v;
    }
    ```

    Then walk the clone; for each plain-object property whose `key.toLowerCase()` is in SENSITIVE:
    - If value is string → replace with `redactPii(value)` if value contains classifiable PII, else with the literal `'[REDACTED]'`.
    - Else (number, boolean, array, object, null) → replace with the literal `'[REDACTED]'`.
    - Continue walking siblings; do NOT recurse into a redacted subtree.

    For non-sensitive keys that hold strings → leave as-is (do NOT auto-run redactPii on every string; that would corrupt non-PII telemetry like `{ "method": "POST" }`).

    For non-sensitive keys that hold objects/arrays → recurse.

    **Defense against prototype pollution (T-01-08):**
    - `Object.create(null)` for the clone container so `__proto__` / `constructor` never resolve.
    - `Object.getPrototypeOf(v) === Object.prototype || === null` guard in `isPlainObject` — refuses to traverse class instances, Maps, Sets, Dates, Buffers, Errors. They are returned as-is (or, if at a sensitive key, replaced with `'[REDACTED]'`).

    **Pure / no-I/O constraint:** This module imports only `node:` builtins or nothing at all. No fs, no fetch, no logging.

    Per D-49 compliance: no regex library, hand-rolled patterns only.

    ```typescript
    import test from 'node:test';
    import assert from 'node:assert/strict';
    import { classifyPii, redactPii, redactKeys } from '../bin/lib/pii.ts';
    import { POSITIVES, NEGATIVES, KEY_FIXTURES } from './fixtures/pii-corpus.ts';

    test('classifyPii: every positive fixture finds at least one matching span of the expected kind', () => {
      for (const c of POSITIVES) {
        const matches = classifyPii(c.input);
        const hit = matches.find(m => m.kind === c.kind && m.raw === c.raw);
        assert.ok(hit, `expected to find ${c.kind}=${JSON.stringify(c.raw)} in ${JSON.stringify(c.input)} — got ${JSON.stringify(matches)}`);
      }
    });

    test('redactPii: every positive fixture, the original raw substring is gone from the output', () => {
      for (const c of POSITIVES) {
        const out = redactPii(c.input);
        assert.ok(!out.includes(c.raw), `expected ${JSON.stringify(c.raw)} to be redacted in: ${out}`);
        assert.ok(out.includes(`[REDACTED:${c.kind}]`), `expected REDACTED:${c.kind} tag in: ${out}`);
      }
    });

    test('classifyPii: negatives produce no matches', () => {
      for (const n of NEGATIVES) {
        const matches = classifyPii(n.input);
        assert.equal(matches.length, 0, `${n.reason}: ${JSON.stringify(n.input)} → ${JSON.stringify(matches)}`);
      }
    });

    test('redactPii is idempotent: redactPii(redactPii(s)) === redactPii(s)', () => {
      for (const c of POSITIVES) {
        const once = redactPii(c.input);
        const twice = redactPii(once);
        assert.equal(twice, once);
      }
    });

    test('redactKeys: top-level sensitive key value replaced', () => {
      const out = redactKeys(KEY_FIXTURES.flat) as { authorization: string; method: string };
      assert.notEqual(out.authorization, 'Bearer sk-abc123');
      assert.equal(out.method, 'POST');
    });

    test('redactKeys: nested sensitive key value replaced (case-insensitive)', () => {
      const out = redactKeys(KEY_FIXTURES.nested) as any;
      assert.notEqual(out.headers['X-API-Key'], 'k_live_xyz');
      assert.equal(out.headers['User-Agent'], 'pensmith/0.1');
    });

    test('redactKeys: recurses into arrays', () => {
      const out = redactKeys(KEY_FIXTURES.array) as any;
      assert.notEqual(out.entries[0].token, 'tok_1');
      assert.notEqual(out.entries[1].token, 'tok_2');
      assert.equal(out.entries[2].method, 'GET');
    });

    test('redactKeys: non-string sensitive values become literal [REDACTED]', () => {
      const out = redactKeys(KEY_FIXTURES.mixed_value_types) as any;
      assert.equal(out.secret, '[REDACTED]');
      assert.equal(out.password, '[REDACTED]');
      assert.equal(out.cookie, '[REDACTED]');
      assert.equal(out.api_key, '[REDACTED]');
    });

    test('redactKeys does not mutate input', () => {
      const input = JSON.parse(JSON.stringify(KEY_FIXTURES.nested));
      const snapshot = JSON.stringify(input);
      redactKeys(input);
      assert.equal(JSON.stringify(input), snapshot);
    });

    test('redactKeys is idempotent on object fixtures', () => {
      const once = redactKeys(KEY_FIXTURES.nested);
      const twice = redactKeys(once);
      assert.deepEqual(twice, once);
    });

    test('redactKeys defends against __proto__ payload (no pollution)', () => {
      const before = ({} as any).polluted;
      redactKeys(JSON.parse('{"__proto__":{"polluted":true}}'));
      const after = ({} as any).polluted;
      assert.equal(after, before, 'Object.prototype must not be polluted');
    });
    ```

    Note the `__proto__` defense test uses `JSON.parse` because object literals don't actually set the prototype. The defense in `isPlainObject` (proto check) and `Object.create(null)` clone container is what makes this safe.
