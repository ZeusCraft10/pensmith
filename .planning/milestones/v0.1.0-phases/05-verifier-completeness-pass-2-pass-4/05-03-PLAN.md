Implement Pass 4 (per-paragraph orphan-claim audit) as an advisory pass whose CORE (claim extraction + orphan detection) is pure-Node deterministic, modeled on `bin/lib/quote-extractor.ts`. Steps 1 and 2 (sentence-level claim extraction and orphan-vs-cited classification) never call an LLM — they implement the PINNED rule R1–R8 authored in Plan 05-01 Task 1 (the same rule the orphan-count fixtures were derived from), per PRD §14. Purpose: VRFY-06. Determinism of claim extraction and orphan detection is the load-bearing correctness property — `tier-contract` parity depends on identical orphan sets on both tiers (and across OSes), which only holds if extraction has zero non-determinism (no NLP library, Pitfall 4). CRITICAL: this plan implements EXACTLY the pinned rule from 05-01; if any orphan-count fixture appears to disagree, the resolution is to re-walk R1–R8 (INCLUDING the R5 8-word floor — a sentence under 8 words is dropped before marker counting) and fix the bug in this extractor — NEVER to count AMBIGUOUS sentences as orphans or lower the HIGH threshold (both violate the non-negotiable). The fixtures were authored from R1–R8 by construction, and every intended claim sentence in them clears R5, so a faithful implementation passes all of them — including the canonical Climate-change example whose three ≥8-word sentences yield orphanCount 1.

      R1 — Sentence split: split a paragraph on the pure regex boundary "a `.`, `!`, or `?` terminator followed by whitespace OR end-of-string". No NLP library, no abbreviation handling (residual abbreviation false-positives are an accepted advisory limitation). Trim each resulting sentence.

"Has a citation within proximity" = a `[@citekey]` token appears within ORPHAN_PROXIMITY_CHARS=500 characters of the sentence's character span inside the paragraph (paragraphs are short, so in practice this is "the paragraph contains a [@citekey]").
      R3 — Rhetorical skip: a sentence ending in `?` is NOT a claim (discarded before marker counting).
      R4 — Definition skip: a sentence matching DEFINITION_MARKERS /\b(defined as|refers to|known as)\b/i is NOT a claim (discarded before marker counting).
      R5 — Length skip: a sentence with word count < CLAIM_MIN_WORDS=8 is NOT a claim. This is a PRE-FILTER applied BEFORE R6 marker counting (it sits in the same skip stage as R3/R4); a sentence dropped at R5 is never marker-counted and can never be a claim or orphan regardless of how many markers it contains. Word count is computed on the trimmed sentence string split on whitespace (no stripCites step — [@citekey] tokens, where present, are counted as words, consistent across fixtures and extractor).
      R6 — Marker counting: count DISTINCT matches of CLAIM_MARKERS /\b(is|are|demonstrates|shows|proves|indicates|suggests|reveals|confirms|establishes|argues|claims|because|therefore|thus|hence|consequently)\b/gi in the surviving sentence. "Distinct" = number of unique marker SURFACE FORMS matched after case-insensitive dedup (e.g. a sentence matching both `proves` and `is` has 2 distinct markers; a sentence matching `is` twice has 1; `is` and `are` are two distinct surface forms and count as 2). To avoid any ambiguity, fixtures MUST NOT rely on `is`/`are` co-occurrence to reach a HIGH classification.
      R7 — Confidence: 0 distinct markers → NOT a claim (dropped); exactly 1 → claimConfidence 'AMBIGUOUS'; 2+ → claimConfidence 'HIGH'.
      R8 — orphanCount (THE deterministic, LLM-INDEPENDENT summary number the fixtures assert): count ONLY 'HIGH'-confidence claim sentences that have NO in-text citation within proximity (R2). AMBIGUOUS sentences are NEVER counted toward orphanCount regardless of any later LLM Step-3 label. This is the non-negotiable: AMBIGUOUS labeling must never change orphanCount.
