<objective>
Implement Pass 2 (claim support) as an advisory side-channel module modeled exactly on `bin/lib/verify/freshness.ts`. For each in-text `[@citekey]`, Pass 2 extracts the citing sentence deterministically, pulls the source's abstract/title/authors from the bib metadata, and (when a live LLM is available) asks the `claim-support` prompt for a verdict in {SUPPORTED, PARTIAL, UNSUPPORTED, UNCLEAR}. Under `PENSMITH_NO_LLM=1` it returns a conservative UNCLEAR placeholder for every citation. It never touches `hasFail`/`status` — that is enforced structurally (this module returns results only; wiring happens in Plan 05-04 below the frozen status line).

Purpose: VRFY-03. The UNCLEAR-bias calibration is the load-bearing correctness property — the prompt + placeholder both default to UNCLEAR rather than manufacturing confident SUPPORTED on thin evidence.
Output: `bin/lib/verify/pass2.ts` exporting `runPass2`, `renderPass2Section`, and the `Pass2Result`/`Pass2Verdict` types. Turns the Plan 05-01 RED behavioral tests in `tests/known-bad-pass2.test.ts` GREEN.
</objective>
